Recognition Guide: Strings and Arrays

Typically you know you are working with a string or array. Use the following guidelines to determine what strategies might be useful.


Two Pointers

Signs to use:
  • Phrases like "find pair", "palindrome", "reverse", "meeting point"
  • The input is a sorted array
  • You need to track two different positions simultaneously
  • Problems involving "opposite ends" or "converging"
Do not use if:
  1. Access to random elements is required
  2. Elements need to be compared with multiple other elements
  3. Elements need to be compared with a different set of elements

Hash Tables

Signs to use:
  • Phrases like "frequency", "count", "unique", and "duplicate"
  • Need to store and retrieve data quickly
  • Need to check for duplicates or frequency of elements
  • Need to store additional information about elements using key-value pairs
Do not use if:
  1. Space complexity is a critical constraint
  2. The original order of elements is important
  3. You're working with simple numeric ranges that can be tracked with array indexing

Sliding Window

Signs to use:
  • Looking for continuous sequences of fixed or variable length that satisfy a specific condition
  • Phrases like "subarray", "substring", "consecutive sequence"
  • Problems involving a "shortest" or "longest" output
  • You need to track a running sum or average
Do not use if:
  1. Elements don't need to be consecutive
  2. The order of elements can be changed

Sorting

Signs to use:
  • Need to find duplicates, the median, or the kth largest element
  • Need to identify a subarray
  • Need to find the smallest or largest element
  • Need to process elements in ascending/descending order
Do not use if:
  1. Order of elements needs to be maintained
  2. Time complexity must be better than O(n log n)
  3. Problem can be solved with a single pass using another technique

Multiple Traversals

Signs to use:
  • Need to compare elements or find a specific element
  • Phrases like "cumulative sum"
  • Problem requires preprocessing
Do not use if:
  1. Real-time processing is required
  2. Time complexity is a critical constraint
  3. Problem can be solved with a single pass using another technique

Traverse from the Right

Signs to use:
  • Need to find the last element that satisfies a condition
  • Phrases like "last", "rightmost", "suffix"
Do not use if:
  1. The order of processing doesn't matter
  2. Forward traversal would work equally well

Pre-computation

Signs to use:
  • Multiple operations are required
  • Need to perform repeated calculations
  • Need to calculate the sum, frequency, range, or maximum/minimum value
  • Need O(1) query time after preprocessing
Do not use if:
  1. Space complexity is a critical constraint
  2. Data is dynamic

Common Problem Patterns

  • Finding pairs: Two pointers or hash table
  • Subarray sums: Sliding window or pre-computation
  • Duplicate detection: Hash table or sorting
  • Range queries: Pre-computation
  • Pattern matching: Sliding window or hash table

Questions to ask in an interview

  1. Can the input be empty?
  2. Can I assume all the inputs will be valid?
  3. Are elements unique or can there be duplicates?
  4. What is the input character set? (a-z, integers, etc)
  5. Is the array sorted?
  6. What's the required time and space complexity?
  7. Does the original order of elements need to be preserved?
