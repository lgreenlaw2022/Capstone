Solution Walkthrough: Convert Sorted Array to Balanced Binary Search Tree

Understand the problem

Test Cases

Output: [0,-3,9,-10,null,5] or [0,-10,5,null,-3,null,9]

Input: nums = [1,3]
Output: [3,1] or [1,null,3]

Input: nums = [1]
Output: [1]

Match

Common solutions to binary tree problems:

  1. Identify the appropriate Tree Traversal: Pre-Order, In-Order, Post-Order, Level-Order: the input is an array so this is not applicable.
  2. Use a binary search tree to find an element or sort a list: we will use a BST
  3. Use definitions of sorted arrays and BSTs: Since we know the data structures for this problem, we need to find a root of the tree that will leave the BST balanced.

Plan

If we pick the median of the array as our root, there will be an equal number of nodes in each subtree. We can recursively find the median of the nodes in the array to dictate the order they are added to the BST. This ensures a balanced tree.

  1. Base case: if the list is empty, return None
  2. recursively find the median of the array
    1. get the index of the new root node, aka the median of the list
    2. build the subtrees by recursively calling the left and right halves of the array
  3. return the root

Complexity Analysis

Time Complexity:

O(n) because there is one recursive call to add a value to the BST.

Space Complexity:

O(n) because we need to store each value from the array as a node in the BST.
