Recognition Guide: Binary Trees

Signs you may want to use binary trees
  1. Use cases of binary trees include graph problems, routing, and decision trees
  2. BSTs are useful in situations where you want efficient search operations
Do not use if:
  1. Storing data in an array would be less complex
Deciding between traversal strategies
  1. Use a depth-first search if you want to explore as far as possible along each branch before backtracking
  2. Use pre-order traversal if you want to copy the tree
  3. Use post-order traversal if you want to delete the tree.
  4. Use in-order traversal if you want to get the sorted order of elements from a BST
  5. Use level order traversal when you want to search for a node that satisfies a specific property

Questions to ask in an interview

  1. Can I assume the tree is balanced or complete?
    • This question is important for clarifying the run time of certain operations
  2. What kind of tree is it? Binary tree? BST?
  3. Can the tree be empty?
  4. Do we know the order in which elements are added to the tree?
    • This can also provide insight into the height of a tree. For example, if you have a mostly sorted array, a BST is going to be right skewed. In this case, it may be better not to use a tree.
  5. Is the input array sorted?
      As described in the previous question, we may opt to not use a tree when building it from a mostly sorted array. Only build BSTs when you need the lookup advantages.
