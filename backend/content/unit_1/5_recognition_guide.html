Recognition Guide: Hash Maps

Signs you may want to use a hash map

  1. You need to store values
  2. You need constant run time but storage is not restricted
  3. You want to count occurrences
Common problems that use hash maps:
  • Finding duplicates
  • Finding a sum
  • Remove the least recently used (LRU) item
  • Mapping one input to another
  • Construct Binary Tree from Preorder and Inorder Traversal

Do not use if:

  1. the order of the data matters

Questions to ask in an interview

  1. Are there any memory constraints?
    • Hash tables can be space-inefficient due to the need for resizing and handling collisions. It's vital to understand if space efficiency is a critical factor.
  2. Do you require a specific time complexity?
    • Asking for specific run time requirements, like needing constant time complexity (O(1)) for insertions, deletions, and lookups, can signal a hash table is appropriate.
  3. Can you describe the nature of the data items you are dealing with?
    • Understanding whether the data items are complex objects, simple key-value pairs, or require relationships between items can help infer the operations needed.
