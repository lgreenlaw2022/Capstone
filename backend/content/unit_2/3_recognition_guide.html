Recognition Guide: Linked Lists

Linked lists are unique in that usually you are told you are working with a linked list. The problem becomes determining which linked list strategy is suited to solving the problem.

Two Pointers

  • use when tracking windows of nodes (sublists)
  • use when positions of certain nodes are important
  • use for linear data structures: input data can be traversed in a linear fashion, such as an array, linked list, or string.
  • use when you want to process data elements at two different positions simultaneously.

Multiple Passes

  • use to traverse two linked lists at the same time
  • use to pre-process the list for other data structures
  • use to determine a characteristic of the input

Temp head

  • important to use if operations might affect the head of the list (eg. deleting, merging, reordering)
  • use when creating a new list

Do not use if:

  1. searching — searching is inefficient because you cannot access items by index
  2. sorting — you cannot access items by index and in cases like a singly linked list, you can only compare pairs of values at a time

Questions to ask in an interview

  1. What type of linked list is it? singly linked, doubly linked?
  2. Could there be cycles in the list?
  3. Will the node values be given in sorted order?
  4. Can I create a new linked list?
    • Some problems require you modify the list in-place. This requires less storage.
  5. What should the method return? A pointer to a head? A brand new list?
  6. What kind of data does a node hold?
    • This might affect how you check for equality between strings, integers, etc.
  7. Does the linked list have a maximum (or minimum) number of nodes?
